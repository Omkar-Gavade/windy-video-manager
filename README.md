# S3 Video Manager

Internal tool to manage short videos generated by Playwright automation and
stored in a private AWS S3 bucket.

The application does exactly four things:

1. **Upload** a video to S3
2. **List** videos from S3
3. **Preview** a video (streamed via presigned URL)
4. **Download** a video (via presigned URL)

Nothing else.

## Architecture

```
React SPA  ‚îÄ‚îÄHTTP/JSON‚îÄ‚îÄ‚ñ∂  FastAPI  ‚îÄ‚îÄboto3‚îÄ‚îÄ‚ñ∂  AWS S3 (private)
```

- The browser never holds AWS credentials.
- The backend is the only component that talks to S3.
- Uploads are proxied through FastAPI (validation + unique naming).
- Preview and download use short-lived presigned URLs (browser ‚Üî S3 directly).

## Environment variables

### Backend (`backend/.env`)

| Variable | Description |
| --- | --- |
| `AWS_REGION` | Bucket region (must match the bucket) |
| `S3_BUCKET` | Target bucket name |
| `S3_PREFIX` | Key prefix the app owns (e.g. `videos/`) |
| `MAX_UPLOAD_MB` | Max accepted upload size |
| `ALLOWED_VIDEO_MIME` | Comma-separated allowed MIME types |
| `PREVIEW_EXPIRY_SECONDS` | Preview presigned URL lifetime |
| `DOWNLOAD_EXPIRY_SECONDS` | Download presigned URL lifetime |
| `ALLOWED_ORIGINS` | Comma-separated CORS origins |

AWS credentials are resolved through the standard boto3 provider chain
(IAM role in production; environment / `~/.aws/credentials` locally). They are
never read by application code.

### Frontend (`frontend/.env`)

| Variable | Description |
| --- | --- |
| `VITE_API_BASE_URL` | Base URL of the FastAPI backend |
| `VITE_MAX_UPLOAD_MB` | Client-side upload size hint (match backend) |

## Local setup

### Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # then edit values
uvicorn app.main:app --reload --port 8000
```

Health check: `GET http://localhost:8000/health`

### Frontend

```bash
cd frontend
npm install
cp .env.example .env   # then edit values
npm run dev
```

App: `http://localhost:5173`

## API endpoints

All responses use a consistent envelope:

- Success: `{ "success": true, "data": <payload> }`
- Error: `{ "success": false, "message": "<safe message>" }`

| Method | Path | Description |
| --- | --- | --- |
| GET | `/health` | Liveness probe |
| GET | `/api/videos` | List all videos in the bucket/prefix |
| POST | `/api/videos/upload` | Upload a video (multipart `file`) |
| GET | `/api/videos/preview?key=<key>` | Presigned inline URL (stream) |
| GET | `/api/videos/download?key=<key>` | Presigned attachment URL (download) |

## Deployment notes

**Backend**

- Run under a production server, e.g.
  `gunicorn -k uvicorn.workers.UvicornWorker app.main:app`.
- Resolve AWS credentials via an **IAM role** (not static keys). The IAM policy
  should be least-privilege: `s3:GetObject`, `s3:PutObject`, `s3:ListBucket`,
  `s3:HeadObject`, scoped to the bucket/prefix only.
- Cap request body size at the reverse proxy / server (e.g. nginx
  `client_max_body_size`) to match `MAX_UPLOAD_MB` ‚Äî this is the pre-buffer
  guard; the app enforces the limit on received bytes as a backstop.
- Set `ALLOWED_ORIGINS` to the deployed frontend origin(s). Serve over HTTPS.
- Ensure `AWS_REGION` matches the bucket region (required for valid presigned
  URLs).

**S3 bucket**

- Keep **Block Public Access** on. Access is only ever via presigned URLs.
- Configure bucket **CORS** to allow the frontend origin for `GET` (and
  `Range`) so the HTML5 player can stream and downloads can start:

  ```json
  [
    {
      "AllowedOrigins": ["https://your-frontend.example"],
      "AllowedMethods": ["GET"],
      "AllowedHeaders": ["*"],
      "ExposeHeaders": ["Content-Length", "Content-Range", "Accept-Ranges"]
    }
  ]
  ```

**Frontend**

- Build with `npm run build`; serve the static `dist/` from any static host
  (S3 + CloudFront, Vercel, nginx).
- Set `VITE_API_BASE_URL` to the deployed backend URL at build time.
